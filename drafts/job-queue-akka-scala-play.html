A job queue for Scala using Akka and Play

As more and more webapps start to integrate machine-learning powered features, the need for asynchronous background processing will only increase. Even for sending emails, apps need a way to manage tasks independently of the HTTP request/response cycle.

On a Python/Django stack, I would have turned to trusty Celery with the recommended RabbitMQ broker backend. For several (good) reasons, however, this project runs on Scala with the Play framework. While I'm happy about the choice, it's true that a few of Django's included batteries are missing in this ecosystem. It turns out that a job queue package is one of those.
